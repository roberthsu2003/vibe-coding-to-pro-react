# Serverless 後端 - 學習主題

## 主題說明

在前端開發時，我們常常需要串接第三方 API（例如 Google Gemini API、OpenAI 等）。
若你直接在 React 前端呼叫 AI API，**API Key 會曝露在瀏覽器的 JavaScript 程式碼中**，任何人都可以在 DevTools 中找到並竊取！

```
// ❌ 危險！這樣做 API Key 會被看見
const ai = new GoogleGenAI({ apiKey: "AIzaSy..." });
```

**Vercel Serverless Functions** 正是解決這個問題最簡單的方法！

> 💡 **什麼是 Serverless？**  
> 你不需要架設、管理、維護伺服器，只需要寫一個 **Function（函式）**，
> 部署到雲端後，它就會在有人呼叫時自動執行，沒人呼叫時不佔用任何資源。

---

## 本單元的目標

將一個由 AI Studio 產生的 **React + TypeScript + Vite** 專案，  
一步一步轉換為使用 **Vercel Serverless Functions** 架構的安全專案。

### 轉換前後比較

| 比較項目 | 轉換前（AI Studio 版） | 轉換後（Serverless 版） |
|---|---|---|
| API Key 位置 | ⚠️ 前端 `vite.config.ts` 注入 | ✅ 伺服器端環境變數 |
| 呼叫 Gemini | ⚠️ 前端直接呼叫 | ✅ 透過 `/api/gemini` 代理 |
| 安全性 | ❌ Key 可被竊取 | ✅ Key 完全不暴露 |
| 部署難度 | 只能在 AI Studio 執行 | ✅ 可部署至 Vercel |

## 系統需求

開始前，請確認以下環境：

- **Node.js**：18.0.0 或以上版本（建議 20 LTS 或更新）
- **npm**：8.0.0 或以上版本
- **Git**：任何版本
- **Vercel 帳號**：免費帳號即可

檢查版本：
```bash
node --version      # 應顯示 v18.0.0 或以上
npm --version       # 應顯示 8.0.0 或以上
```

## 快速參考：常用指令

### 本機開發

```bash
# 進入專案目錄
cd my-serverless-app

# 安裝依賴
npm install

# 方式 A：啟動雙伺服器（理解 proxy 運作原理）
npm run dev           # 終端機視窗 1：Vite（port 3000）
vercel dev --listen 3001  # 終端機視窗 2：Vercel CLI（port 3001）

# 方式 B：單一伺服器（推薦，自動設定 proxy）
vercel dev            # 一個指令搞定（http://localhost:3000）
```

### 部署到 Vercel

```bash
# 方式 A：透過 GitHub 自動部署（推薦）
git push              # 推送到 GitHub，Vercel 自動部署

# 方式 B：手動部署到預覽環境
vercel

# 方式 C：手動部署到正式環境
vercel --prod
```

<details>
<summary>💡「預覽環境」和「正式環境」有什麼不同？</summary>

- **預覽環境 (`vercel`)**：將目前電腦裡的程式碼打包上傳，產生一個臨時的隨機網址（例如 `https://app-random123.vercel.app`）。你可以自由地測試新功能，這**絕對不會影響到一般使用者正在瀏覽的正式網站**，就像上台前的「彩排」。
- **正式環境 (`vercel --prod`)**：直接更新你對外公開的主要網址（例如 `https://app.vercel.app`）。所有的使用者都會立刻看到最新版本，這就是真實的「正式演出」。

</details>

### 環境變數管理

```bash
# 設定環境變數
vercel env add GEMINI_API_KEY

# 查看環境變數清單
vercel env list

# 拉取遠端環境變數到本機 .env.local
vercel env pull

# 查看日誌（排除問題）
vercel logs --follow
```

<details>
<summary>💡 指令詳細說明</summary>

- **`vercel env add TEAM_API_KEY`**：會以互動式問答引導你將機密的 API Key 設定在上線環境，確保安全。
- **`vercel env list`**：列出目前 Vercel 專案上已設定了哪些環境變數的名稱（不會顯示金鑰明碼，保障安全）。
- **`vercel env pull`**：這招非常好用！它會自動把你 Vercel 雲端上所有的環境變數抓下來，安全地存在本機的 `.env.local` 檔案裡。適合換電腦或團隊協作時一次搞定本機測試設定。
- **`vercel logs --follow`**：當線上網站出現問題（例如 500 錯誤）時，這個指令能即時顯示 Serverless Function 此刻報出的詳細錯誤訊息（Error Log），是抓蟲調試的最佳幫手。

</details>

## 專案架構圖

### 轉換後的目錄結構

```
gemini-一句話問答/        ← 你操作的專案根目錄
│
├── api/                  ← 🆕 新增！Serverless Functions 放這裡
│   └── gemini.ts         ← Vercel 自動識別為 /api/gemini 端點
│
├── src/
│   ├── App.tsx           ← ✏️ 需修改：改為呼叫 /api/gemini
│   ├── main.tsx
│   └── index.css
│
├── .env.local            ← 🆕 新增！存放本機用的 API Key（不上傳 Git）
├── .env.example          ← ✏️ 需更新：移除注入至前端的方式說明
├── .gitignore            ← ✏️ 需確認：確保 .env.local 不會被上傳
├── vite.config.ts        ← ✏️ 需修改：移除 define、加入 proxy 設定
├── package.json          ← ✏️ 需修改：移除 @google/genai，加入 @vercel/node
└── tsconfig.json
```

### 資料流向圖

```
使用者瀏覽器
    │  (1) 填入問題，點選「送出提問」
    ▼
前端 App.tsx
    │  (2) POST /api/gemini   { prompt: "..." }
    ▼
Vercel Serverless Function（api/gemini.ts）
    │  (3) 讀取 process.env.GEMINI_API_KEY（伺服器端，前端看不到）
    │  (4) 呼叫 Google Gemini API
    ▼
Google Gemini API
    │  (5) 回傳 AI 回答
    ▼
Serverless Function 將結果回傳給前端
    │  (6) { text: "..." }
    ▼
前端顯示 AI 回答
```
