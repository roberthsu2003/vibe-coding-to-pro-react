# 範例 2：修改前端呼叫方式

## 目標

在上一個範例中，我們建立了 Serverless Function（`api/gemini.ts`）來安全地處理 Gemini API 呼叫。  
現在我們要修改 **前端的 `src/App.tsx`**，讓它改為呼叫我們自己的 `/api/gemini`，  
而不是直接呼叫 Google Gemini API。

---

## 理解目前的問題

打開 `src/App.tsx`，找到 `handleSubmit` 函式中的這段程式碼：

```typescript
// ❌ 目前的做法（有安全疑慮）
import { GoogleGenAI } from "@google/genai";  // ← 前端直接引入 SDK

const handleSubmit = async (e: FormEvent) => {
  const apiKey = process.env.GEMINI_API_KEY;  // ← Key 被注入到前端！
  const ai = new GoogleGenAI({ apiKey });
  const result = await ai.models.generateContent({
    model: "gemini-3-flash-preview",
    contents: prompt,
  });
  setResponse(result.text || "AI 沒有回傳內容。");
};
```

**問題點：**
- `process.env.GEMINI_API_KEY` 的值在 build 時被 Vite 直接嵌入 JS 檔案
- 任何人在 Chrome DevTools 都可以找到你的 Key
- 前端直接相依 `@google/genai` SDK，增加打包體積

---

## 步驟 1：修改 `src/App.tsx`

打開 `src/App.tsx`，進行以下兩項修改：

### 1-A：移除不需要的 import

找到這一行並**刪除**：

```typescript
// 刪除這行
import { GoogleGenAI } from "@google/genai";
```

### 1-B：修改 `handleSubmit` 函式

將整個 `try...catch` 區塊替換為以下內容：

```typescript
// ✅ 修改後的做法（安全）
const handleSubmit = async (e: FormEvent) => {
  e.preventDefault();
  if (!prompt.trim()) return;

  setLoading(true);
  setError("");
  setResponse("");

  try {
    /**
     * ✅ 前端只呼叫我們自己的 Serverless Function（/api/gemini）
     * GEMINI_API_KEY 完全不會出現在前端程式碼或瀏覽器中
     */
    const res = await fetch("/api/gemini", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ prompt }),  // ← 只傳送問題文字
    });

    if (!res.ok) {
      throw new Error(`伺服器回應錯誤：${res.status}`);
    }

    const data = await res.json();
    setResponse(data.text || "AI 沒有回傳內容。");
  } catch (err: any) {
    console.error(err);
    setError("呼叫 Gemini 失敗，請確認網路連線或伺服器設定。");
  } finally {
    setLoading(false);
  }
};
```

---

## 修改前後對照

| | 修改前 | 修改後 |
|---|---|---|
| **import** | `import { GoogleGenAI } from "@google/genai"` | 無（不需要） |
| **呼叫方式** | `ai.models.generateContent(...)` | `fetch("/api/gemini", {...})` |
| **API Key** | `process.env.GEMINI_API_KEY`（前端讀取） | 完全不需要在前端處理 |
| **安全性** | ❌ Key 暴露於瀏覽器 | ✅ Key 只在伺服器端 |
| **程式碼複雜度** | 需要 SDK 知識 | 只需 `fetch` API |

## 完整修改後的 `App.tsx`

以下是完整的、修改完成後的 `App.tsx` 樣子（供對照參考）：

```tsx
// src/App.tsx（修改完成版）
import { useState, FormEvent } from "react";
import { motion, AnimatePresence } from "motion/react";
import { Send, Loader2, Sparkles, AlertCircle } from "lucide-react";
// ✅ 已移除 @google/genai 的 import

export default function App() {
  const [prompt, setPrompt] = useState("");
  const [response, setResponse] = useState("");
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState("");

  const handleSubmit = async (e: FormEvent) => {
    e.preventDefault();
    if (!prompt.trim()) return;

    setLoading(true);
    setError("");
    setResponse("");

    try {
      // ✅ 呼叫自己的 Serverless Function，不再直接呼叫 Gemini
      const res = await fetch("/api/gemini", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ prompt }),
      });

      if (!res.ok) {
        throw new Error(`伺服器回應錯誤：${res.status}`);
      }

      const data = await res.json();
      setResponse(data.text || "AI 沒有回傳內容。");
    } catch (err: any) {
      console.error(err);
      setError("呼叫 Gemini 失敗，請確認網路連線或伺服器設定。");
    } finally {
      setLoading(false);
    }
  };

  return (
    // ... JSX 部分不需要修改（保持原樣）
  );
}
```

> 📝 **提示**：JSX 的部分（`return` 以下的 HTML 結構）完全不需要修改，  
> 只有 `import` 和 `handleSubmit` 函式需要更動。

## 為什麼這樣更好？

### 安全性對比

```
修改前：
瀏覽器 → 前端 JS（含 API Key）→ Google Gemini API
              ↑
        ❌ 任何人都能在這裡找到你的 Key

修改後：
瀏覽器 → 前端 JS → /api/gemini（Serverless Function）→ Google Gemini API
                          ↑
                    ✅ API Key 只在這個函式裡，前端看不到
```

### 職責分離

| 層級 | 職責 |
|---|---|
| **前端 App.tsx** | 顯示 UI、收集使用者輸入、呼叫 `/api/gemini` |
| **Serverless Function** | 持有 API Key、呼叫 Gemini API、回傳結果給前端 |
