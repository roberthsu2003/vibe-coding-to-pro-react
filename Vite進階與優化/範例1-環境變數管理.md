# 範例 1：環境變數管理 (.env)

## 目標

學會在 Vite 專案中設定環境變數，分離開發機/正式機的 API 網址，並了解其安全限制。

---

## 步驟 1：觀察環境變數設定檔

進入 `vite-advanced-demo` 專案目錄，您會看到兩個環境變數檔案：

1. `.env.development`：開發階段 (`npm run dev`) 會讀取的變數
2. `.env.production`：產品建置 (`npm run build`) 時會讀取的變數

打開 `.env.development`，可以看到：
```env
VITE_API_URL=http://localhost:3000/api
# ...其他變數
```

> **⚠️ 重大資安提醒 (非常重要)**：只有以 `VITE_` 開頭的變數，Vite 才會將它打包並暴露給前端程式碼。**絕對不可以**將 `GEMINI_API_KEY` 等敏感金鑰加上 `VITE_` 前綴！若需要安全地隱藏金鑰，我們將在後續的 [BFF + Express API Proxy 後端代理架構](../BFF+Express_API_proxy/README.md) 章節中學習如何實作。

---

## 步驟 2：在前端程式中讀取變數

開啟 `vite-advanced-demo/src/App.tsx`。

尋找這行程式碼：
```typescript
const apiUrl = import.meta.env.VITE_API_URL;
```
Vite 使用 `import.meta.env` 物件來提供環境變數。

---

## 步驟 3：啟動並驗證

在終端機執行指令：
```bash
cd vite-advanced-demo
npm run dev
```
打開瀏覽器，觀察畫面或是開發人員工具中的 `console.log`，確認能正確拿到 `.env.development` 中的 `VITE_API_URL` 網址。

> **⚠️ 重大資安提醒：** 絕對不要把 Gemini API Key 或 OpenAI Key 直接寫在前端的 `.env` 裡！這些 Key 一旦跟著打包送到瀏覽器，將可被任何人看見並盜刷（解法請參考 [BFF 架構](../BFF+Express_API_proxy/README.md)）。前端的 `VITE_` 變數通常只用來儲存安全的 API 進入點網址。

---

## 完成

您已學會如何安全地設定和使用環境變數。
