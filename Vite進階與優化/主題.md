# Vite 進階配置與優化

## 主題說明

在先前的「Vite入門」章節中，我們學會了如何啟動和打包專案，並認識了基本的 `vite.config.ts`。但當專案規模擴大為產品級應用時，我們會面臨多個挑戰：環境變數管理、深層資料夾導致的路徑地獄、後端 API CORS 問題、以及打包檔案過大。

本單元將手把手帶您解決這四個常見的開發痛點，幫助您把專案從「玩具」晉升為「企業級應用」。

### 學習目標

- 安全地設定與讀取環境變數 (`.env`)
- 設定路徑別名 (Aliases) 簡化 `import` 路徑
- 使用 Vite 的跨域代理 (Development Proxy) 解決 CORS
- 進行建置優化與第三方套件 Chunk 分割

### 範例專案環境準備

本章提供了一個完整的範例專案，裡面已經寫滿了各種進階配置的詳細註解。這將是我們接下來所有範例的起點。
👉 **[點此進入完全配置好的 vite-advanced-demo 範例資料夾zip檔案](./vite-advanced-demo.zip)**

在開始學習前，請先準備好範例專案的環境，打開您的終端機並執行：

```bash
cd Vite進階與優化/vite-advanced-demo
npm install
```
環境準備好之後，請依照下列步驟**1步1步**跟著操作，每個範例都會帶您修改及了解專案內的設定。

### 學習步驟指引（本章範例）

1. **第一步**：[範例 1：環境變數管理](範例1-環境變數管理.md) (學習分離開發與正式環境變數)
2. **第二步**：[範例 2：路徑別名 (Aliases)](範例2-路徑別名.md) (解決深層 import 路徑地獄)
