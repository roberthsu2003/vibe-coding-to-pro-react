# 範例 4：編譯整合與雲端部署準備

## 目標

前面我們成功用兩個終端機跑起了前面的 React 和後面的 Express。
現在，為了獲得更爽快的開發體驗，並且為未來能順利丟上 Render 或其他雲端平台做準備，我們要做最後一個重要的步驟。

我們需要：
1. 利用 `concurrently` 讓我們只要敲一行指令，就能同時啟動兩個伺服器。
2. 讓 Express 知道，當這套程式到了正是機運作 (Production) 的時候，除了提供 API，還要學會怎麼把 Vite 打包好的前端靜態網頁（`dist` 資料夾）當作檔案伺服器送給使用者。

---

## 步驟 1：使用 concurrently 同時啟動前後端

請打開專案根目錄下的 `package.json` 的 `"scripts"` 區塊。

我們新增一組一鍵啟動指令 `"dev:all"`，讓它用 `concurrently` 同時去跑 Vite 開發伺服器和 Express 開發伺服器：

```json
  "scripts": {
    "dev": "vite",
    "dev:server": "tsx server/server.ts",
    "dev:all": "concurrently \"npm run dev\" \"npm run dev:server\"",  // 增加這行！
    "build": "tsc -b && vite build",
    "lint": "eslint .",
    "preview": "vite preview"
  },
```

設定好之後，未來你只要在終端機輸入：

```bash
npm run dev:all
```

就會看到前端 (`5173`) 跟後端 (`3000`) 的啟動訊息被完美地印在同一個黑色窗格裡，不用再為開視窗煩惱了！

---

## 步驟 2：後端提供靜態檔案服務 (Production 部署用)

當你準備要將這個專案部署上線時，你不會使用 `npm run dev`，而會先把它構建成正式的產品：
```bash
npm run build
```
此時，Vite 會把所有的 React 程式碼壓縮成最佳化的瀏覽器靜態檔案，並且放到一個名為 `dist` 的資料夾中。

線上環境的雲端主機通常**只有一個伺服器在運作**。這時候就是我們的 `server.ts` 出馬的時刻！我們的 Express 必須一肩扛起兩個責任：
1. 繼續服務 `/api/chat` 的代理工作。
2. 對於其他非 API 的普通網頁請求，就把 `dist` 裡面的那些靜態檔案回傳給使用者。

請打開 `server/server.ts`，並在原本的 API 下方加入最後一段設定：

```typescript
import express from 'express';
import cors from 'cors';
import dotenv from 'dotenv';
import { GoogleGenAI } from '@google/genai';
import path from 'path';        // 用來處理好朋友資料夾路徑，請確保你有引入它並在專案安裝。

dotenv.config();

const app = express();
const PORT = process.env.PORT || 3000;

app.use(cors());
app.use(express.json());

app.get('/api/status', (req, res) => {
  res.json({ message: 'BFF 伺服器運作正常！', status: 'OK' });
});

app.post('/api/chat', async (req, res) => {
  // 原本的程式並未更動...
  try {
    if (!process.env.GEMINI_API_KEY) {
      return res.status(500).json({ error: '找不到 GEMINI_API_KEY 環境變數設定。' });
    }
    const ai = new GoogleGenAI();
    const userMessage = req.body.message;
    if (!userMessage) return res.status(400).json({ error: '請提供 message 欄位。' });

    const response = await ai.models.generateContent({
      model: 'gemini-2.5-flash',
      contents: userMessage,
    });

    res.json({ text: response.text });
  } catch (error) {
    console.error('呼叫 Gemini 發生錯誤:', error);
    res.status(500).json({ error: '伺服器內部發生錯誤，請稍後再試。' });
  }
});

// ------------- 部署靜態檔案支援 -------------
// 當這行被啟動，所有的靜態請求都會嘗試到 'dist' 資料夾裡找檔案
app.use(express.static(path.join(__dirname, '../dist')));

// 由於 React 是單頁應用 (SPA)，如果使用者亂打網址或是重新整理
// Express 必須統一回傳 index.html，把路由的決定權交環給 React Router 或是 React 處理。
app.get('*', (req, res) => {
  res.sendFile(path.join(__dirname, '../dist/index.html'));
});
// ------------------------------------------

app.listen(PORT, () => {
  console.log(`[Server] Express 伺服器已啟動: http://localhost:${PORT}`);
});
```

現在，如果要部署到譬如 Render 的雲端平台，你只需要在裡面設定：
- **Build 指令**：`npm install && npm run build`
- **Start 指令**：`npm run dev:server` (不過正式版我們通常會先編譯 TypeScript 再用 raw node 跑，這裡為了教學單純，我們可以直接用 tsx 啟動)
- **環境變數**：在後台將 `GEMINI_API_KEY` 設定進去。
