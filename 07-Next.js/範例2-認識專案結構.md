# 範例 2：認識專案結構

## 目標

認識 Next.js App Router 專案的資料夾與檔案結構。

---

## 前置條件

- 已完成 [範例 0：建立專案](範例0-建立專案.md)

---

## 步驟 1：專案根目錄

```
examples/
├── src/
│   └── app/          ← 應用程式（App Router）
├── public/           ← 靜態資源
├── package.json
├── next.config.ts    ← Next.js 設定
└── tsconfig.json
```

---

## 步驟 2：app 資料夾（建立專案後的初始結構）

完成範例 0 後，`create-next-app` 預設會產生：

```
src/app/
├── layout.tsx        ← 根佈局（共用 HTML 結構）
├── page.tsx          ← 首頁（/）
├── globals.css       ← 全域樣式
└── favicon.ico
```

> **說明**：`about`、`products`、`contact` 會在範例 3～5 中逐步加入。

---

## 步驟 3：重要檔案說明

| 檔案 | 說明 |
|------|------|
| **layout.tsx** | 根佈局，包住所有頁面，可放導覽列 |
| **page.tsx** | 對應路由的頁面，`app/page.tsx` = `/` |

---

## 步驟 4：檔案即路由

在 `app` 下建立資料夾並在資料夾內加入 `page.tsx`，即自動成為路由：

| 路徑 | URL |
|------|-----|
| `app/page.tsx` | `/` |
| `app/about/page.tsx` | `/about` |
| `app/products/page.tsx` | `/products` |

> **說明**：本章節的 `examples` 資料夾為**最原始的基本 template**，請依序完成範例 3～5，依照教學修改檔案，逐步加入 About、Products、Contact 頁面。

---
