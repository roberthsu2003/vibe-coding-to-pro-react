# 範例：靜態三件套（計數器）

## 目的

用一個極小的頁面體驗：**改程式 → 存檔 → 手動重新整理瀏覽器** 的開發迴圈，並以 **ES Module**（`import`／`export`）拆開邏輯與進入點。

## 檔案說明

| 檔案 | 角色 |
|------|------|
| `index.html` | 以 `<script type="module" src="./main.js">` 載入模組 |
| `main.js` | 進入點：綁定 DOM、從 `counter.js` 匯入函式 |
| `counter.js` | 計數狀態與 `getCount`、`increment` 的匯出 |

## 操作步驟

本範例使用 **跨檔案 `import`**，若以 `file://` 直接雙擊開啟 `index.html`，部分瀏覽器會阻擋模組載入。請在 `靜態頁面` 目錄啟動本機靜態伺服器後，用 **http** 開啟：

```bash
cd 靜態頁面
npx --yes serve .
```

終端機會顯示本機網址（常見為 `http://localhost:3000`），用瀏覽器開啟該網址即可。

1. 點擊「+1」按鈕，確認數字會增加。
2. 修改 `counter.js`（例如將 `count` 初值改為 `10`）或 `main.js`，**存檔後**按瀏覽器重新整理，觀察變化。

## 對照思考

- ES Module 需明確路徑與 `type="module"`；拆成多檔後，**本機靜態伺服器**是常見做法（與後續 Vite dev server 概念銜接）。
- 沒有型別檢查，改名或傳錯參數要到執行時才容易發現。
