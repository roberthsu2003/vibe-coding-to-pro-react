
# 10 - Next.js × Vercel 雲端原生功能

## 主題說明

本單元以 **Next.js 16** 為框架，介紹 Vercel 平台的五項雲端原生功能。

這些功能代表了現代網頁開發從「自己管伺服器」轉向「雲端原生（Cloud Native）」的趨勢——  
把 Next.js 部署到 Vercel 後，複雜的後端問題（檔案儲存、快取、效能優化、邊緣運算）都已被平台原生解決。  
你只需要寫少量程式碼，即可獲得業界等級的基礎設施。

---

### 學習目標

- 理解 **Route Handlers** 如何自動成為 Serverless Functions，掌握 App Router 的 API 寫法
- 分辨 Server Component、Server Action、Route Handler 都屬於伺服器端程式，敏感 token 不一定只能放在 `/api`
- 使用 **`<Image>`** 元件實現圖片自動優化（WebP 轉換、Lazy Load、防止版面位移）
- 了解 **Edge Runtime** 的使用時機、優勢與限制，與 Node.js Runtime 的差異
- 使用 **Vercel Blob** 儲存使用者上傳的檔案，取得公開 URL
- 使用 **Upstash Redis**（取代已停售的 Vercel KV）實作計數器與 Rate Limiting

> **重要觀念：保護 token 不等於一定要寫 `/api`**  
> 在 Next.js（React 全端框架）App Router 中，Server Component（伺服器元件）、Server Action（伺服器動作）與 Route Handler（路由處理函式）都可以在伺服器端執行，都可以安全讀取環境變數。  
> 本章有些範例使用 `/api`，是因為它們需要提供可被瀏覽器 `fetch()` 或外部系統呼叫的 HTTP（超文字傳輸協定）端點；如果只是伺服器端讀資料後產生頁面，通常可以直接寫在 Server Component，不必先繞一層自己的 API。

### 練習專案建議（一次建立、全章共用）

本章五個範例建議都在同一個 Next.js 練習專案中完成（避免每個範例都重建專案、重設環境變數）。

> 範例 1 會帶你建立專案，你也可以先在開始前自行建立：

```bash
npx create-next-app@latest cloud-features --typescript --tailwind --app --src-dir --no-import-alias
cd cloud-features
```

這行指令會在目前資料夾底下建立 `cloud-features/`，並初始化一個適合本章練習的 Next.js（React 全端框架）專案。建立完成後，使用 `cd cloud-features` 切換到新專案資料夾，後續才會在正確位置執行 `npm install`、`npm run dev` 等指令。

<details>
<summary>指令與參數說明（可展開）</summary>

- `npx`：使用 npm（Node.js 套件管理器）提供的套件執行器，臨時執行 CLI（命令列介面）工具，不需要先全域安裝。
- `create-next-app@latest`：執行 Next.js（React 全端框架）官方建立專案工具，`@latest` 代表使用目前最新版。
- `cloud-features`：新專案的資料夾名稱，指令完成後會產生 `cloud-features/`。
- `--typescript`：建立 TypeScript（JavaScript 型別擴充語言）版本的專案，讓程式碼有型別檢查。
- `--tailwind`：安裝並初始化 Tailwind CSS（工具類優先 CSS 框架），方便本章快速建立介面樣式。
- `--app`：使用 App Router（Next.js 檔案式路由系統），也就是以 `app/` 目錄組織頁面、版面與 API 路由。
- `--src-dir`：把主要原始碼放在 `src/` 目錄下，例如 `src/app/...`，讓專案根目錄保持乾淨。
- `--no-import-alias`：不設定 `@/*` 這類 import alias（匯入路徑別名），教材會先使用相對單純的路徑結構。

</details>
