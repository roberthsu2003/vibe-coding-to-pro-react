# 範例 5：聯合型別與型別窄化

## 目標

學會**聯合型別**（`|`）、**字面量型別**，以及透過條件判斷進行**型別窄化**（narrowing），讓 TypeScript 在分支裡推斷出更精確的型別。

---

## 前置條件

- 已完成 [範例 4：陣列與泛型入門](範例4-陣列與泛型入門.md)

---

## 步驟 1：聯合型別（`A | B`）

變數允許**多種型別**時，用 **`|`** 連接：

```typescript
type Id = string | number

let value: Id
value = 'user-1'
value = 1001
// value = true  // ❌ boolean 不在 Id 內
```

常見用途：後端可能回傳數字 ID 或字串 UUID、表單欄位可能是字串或數字等。

---

## 步驟 2：字面量型別

不只有 `string`，還可以限定**只能是某幾個固定值**：

```typescript
type Theme = 'light' | 'dark'
type Size = 'S' | 'M' | 'L'

const theme: Theme = 'dark'
// const bad: Theme = 'blue'  // ❌ 不在聯合裡
```

與 `string` 相比，字面量聯合能避免拼錯魔術字串。

---

## 步驟 3：為什麼需要「型別窄化」？

當變數是 `string | number` 時，TypeScript **不允許**直接當成其中一種用：

```typescript
function printLength(x: string | number): void {
  // console.log(x.length)  // ❌ number 沒有 .length（在此分支不成立）
}
```

必須在程式裡**縮小範圍**（窄化），讓編譯器確定此時的型別。

---

## 步驟 4：用 `typeof` 窄化

```typescript
function printLength(x: string | number): void {
  if (typeof x === 'string') {
    console.log(x.length)  // 此區塊內 x 為 string
  } else {
    console.log(x.toFixed(2))  // 此區塊內 x 為 number
  }
}
```

---

## 步驟 5：用 `===` / `!==` 與 `null`／`undefined`

常見寫法：先排除 **`null`** 或 **`undefined`**，再使用其餘成員：

```typescript
function getLength(s: string | null): number {
  if (s === null) {
    return 0
  }
  return s.length  // 此處 s 已窄化為 string
}
```

可選鏈與空值合併在實務上也很常用：`s?.length ?? 0`（語意略有不同，可依需求選擇）。

---

## 步驟 6：可區辨聯合（Discriminated Union）

物件的**共同欄位**（常命名為 `kind`、`type`、`status`）帶有**字面量標籤**，可用 `switch` 窄化：

```typescript
type Success = { ok: true; data: string }
type Failure = { ok: false; error: string }
type Result = Success | Failure

function handle(result: Result): void {
  if (result.ok) {
    console.log(result.data)  // Success
  } else {
    console.log(result.error) // Failure
  }
}
```

這種模式在處理 API 回傳、表單狀態時非常常見。

---

## 步驟 7：`in` 運算子（選讀）

若兩種物件**沒有共同的可區辨字面量**，可檢查屬性是否存在：

```typescript
type Cat = { meow: () => void }
type Dog = { bark: () => void }

function speak(pet: Cat | Dog): void {
  if ('meow' in pet) {
    pet.meow()
  } else {
    pet.bark()
  }
}
```

## 步驟 8：在練習專案中試作

在 `src/main.ts`：

```typescript
type LoadState =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'success'; items: string[] }
  | { status: 'error'; message: string }

function describe(state: LoadState): string {
  switch (state.status) {
    case 'idle':
      return '尚未開始'
    case 'loading':
      return '載入中…'
    case 'success':
      return `共 ${state.items.length} 筆`
    case 'error':
      return `錯誤：${state.message}`
  }
}

console.log(describe({ status: 'success', items: ['a', 'b'] }))
```

執行 `npm run dev`，在瀏覽器主控台查看輸出。

---

## 重點整理

| 概念 | 說明 |
|------|------|
| `A \| B` | 聯合：值可以是 A 或 B |
| `'a' \| 'b'` | 字面量聯合：只能是這幾個字串之一 |
| 型別窄化 | 用 `if`、`switch`、`typeof`、`in` 等縮小型別範圍 |
| 可區辨聯合 | 用共同欄位（如 `status`）+ 字面量區分變體 |

---

## 小練習

1. 定義 `type Role = 'guest' | 'user' | 'admin'`，寫函式 `canEdit(role: Role): boolean`（例如僅 `admin` 為 `true`）。  
2. 寫函式 `addOne(x: number | string)`：若為 `number` 回傳 `x + 1`；若為 `string` 則先 `Number(x)` 再加 1（進階：處理 `NaN`）。  
3. 建立 `type ApiResult = { ok: true; value: number } | { ok: false; reason: string }`，用 `if (result.ok)` 分別存取 `value` 與 `reason`。
