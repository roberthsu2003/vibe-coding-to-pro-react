# 08 - TypeScript 語法

## 主題說明

本單元專門講解 **TypeScript 語法**：變數與型別註解、函式、介面、泛型、聯合型別、工具型別等，讓你在閱讀與撰寫 React / Vite 專案時能對型別系統有完整概念。

對象為已讀過 [03 - TypeScript 配置](../03-TypeScript配置/主題.md)（tsconfig、`.ts` / `.tsx`）的初學者；本單元**不**重複專案設定，聚焦在**程式語法**。

### 練習原則（本單元所有範例適用）

講義中的程式碼區塊，以及本 repo 內 [`tsx-cli/src/`](tsx-cli/src/)、[`examples/`](examples/) 裡附帶的檔案，**僅供對照與參考**。**請在你自己的練習檔**（依 [範例 0](範例0-建立練習專案.md) 建立的專案，或各範例的 `practice-….ts` 等）中，**手動輸入**或**從講義複製**程式後再執行。**不要**以「只執行教材已寫好的示範檔、自己不撰寫程式」當作完成學習。

> 第一次來？請先依 [範例 0](範例0-建立練習專案.md) 準備環境，再在**自己的練習檔**依 [範例 1](範例1-變數與基本型別.md) 輸入或貼上程式，最後用終端機執行該檔案。下列以本 repo 的 `tsx-cli` 路徑為例：

```bash

cd 08-TypeScript語法/tsx-cli

```

```bash
npx tsx src/01-basics.ts
```

（寫法說明見下方「手動建立指令列練習環境」→ **步驟 5** → **方法 1**。）

確認終端機輸出符合預期後，再依序完成範例 2～6（同樣以**自己撰寫／貼上**的程式為主，勿只跑現成檔案）。

## 手動建立指令列練習環境（tsx）

若你**沒有**本 repo 的 `08-TypeScript語法/tsx-cli` 資料夾，或想在**任意路徑**自建一個與講義相同的練習專案，可依下列步驟手動安裝 **[tsx](https://github.com/privatenumber/tsx)**（在終端機直接執行 `.ts` 的工具；與 React 副檔名 `.tsx` 不同，請勿混淆）。

### 步驟 1：建立資料夾並初始化 npm

```bash
mkdir my-ts-practice
cd my-ts-practice
npm init -y
```

### 步驟 2：安裝開發依賴

```bash
npm install -D tsx typescript
```

- **`tsx`**：執行 TypeScript／ESM，無需先編譯成 `.js`。  
- **`typescript`**：讓編輯器與 `tsc` 能做型別檢查（本練習以 `tsx` 跑檔案為主）。

### 步驟 3：設定 `package.json`

用編輯器開啟 `package.json`，至少加入 **`"type": "module"`**（與本教材 `tsx-cli` 一致，使用 ES 模組），並可自行加入 **`scripts`**：

```json
{
  "name": "my-ts-practice",
  "private": true,
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "01": "tsx src/01-basics.ts",
    "watch:01": "tsx watch src/01-basics.ts"
  },
  "devDependencies": {
    "tsx": "^4.19.2",
    "typescript": "~5.7.2"
  }
}
```

> 版本號可依 `npm install` 後實際寫入為準；`scripts` 可照 [本 repo 的 tsx-cli `package.json`](tsx-cli/package.json) 複製 `01`～`06` 與 `watch:*`。

### 步驟 4：新增 `tsconfig.json`

在專案根目錄建立 `tsconfig.json`（可與講義 [tsx-cli/tsconfig.json](tsx-cli/tsconfig.json) 相同，讓行為一致）：

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "skipLibCheck": true,
    "noEmit": true,
    "esModuleInterop": true,
    "verbatimModuleSyntax": true
  },
  "include": ["src"]
}
```

### 步驟 5：建立練習檔並執行

```bash
mkdir -p src
```

在 `src/01-basics.ts` 寫入一行測試：

```typescript
console.log("Hello from tsx");
```

執行方式可擇一（依習慣與是否已 `npm install`）：

### 方法 1：使用 npx（推薦）

```bash
npx tsx your-file.ts
```

將 `your-file.ts` 換成你的檔案路徑即可。本教材在 `tsx-cli` 內的對應範例為：

```bash
npx tsx src/01-basics.ts
```

首次在某台電腦使用時，`npx` 可能會下載 **tsx**；若該專案已執行過 `npm install`，通常會優先使用 `node_modules` 內已安裝的版本。

### 方法 2：使用 npm scripts（已設定 `scripts` 時）

```bash
npm run 01
```

### 方法 3：存檔後自動重跑（watch）

```bash
npm run watch:01
```

### 與本教材對齊

若要與講義範例 1～6 的**檔名與指令**一致，可在 `package.json` 的 `scripts` 補上 `01`～`06`（見 [tsx-cli/README.md](tsx-cli/README.md)）。**學習時**請以講義步驟在**自己的** `src/*.ts` 撰寫程式；本 repo 內同名檔案可作**完成後對照**之用，不宜取代自行練習。

## 學習目標

- 熟練使用型別註解與基本型別
- 為函式參數與回傳值標註型別
- 使用 `interface` 與 `type` 描述物件與結構
- 認識陣列、泛型與常見集合型別
- 理解聯合型別、型別窄化與實務常用寫法
- 使用工具型別（`Partial`、`Pick`、`Omit`）與型別斷言（`as`、`as const`）

---

## 學習流程

每個範例的建議學習方式：

1. **閱讀講義 `.md`** — 了解概念與程式碼範例  
2. **在自己的練習檔撰寫程式** — 手打或從講義複製到 `tsx-cli` 風格專案或 `practice-….ts`，**不要**只執行 repo 內現成檔案  
3. **執行自己的檔案** — `npx tsx …` 或 `npm run 0N`，比對輸出是否正確；必要時再對照本 repo [`tsx-cli/src/`](tsx-cli/src/) 的**參考解答**  
4. **開啟 examples 網頁版**（選用）— 各範例為獨立主題介面，呼應相同觀念；在 `practice-….ts` 改寫並 `npm run dev` 預覽
