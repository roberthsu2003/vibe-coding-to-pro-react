## 步驟 1：`let` 與 `const`

在**目前使用的練習檔**中輸入（或貼上後再改）：

| 關鍵字 | 說明 |
|--------|------|
| **`const`** | 綁定後**不可重新指定**另一個值（常數綁定）。物件／陣列的「內容」仍可修改（除非用其他方式凍結）。 |
| **`let`** | 可重新指定，作用域為**區塊**（`{ }` 內）。 |

```typescript
const pi = 3.14;
// pi = 3  // ❌ 錯誤：const 不可重新賦值

let score = 0;
score = 85; // ✅ 可以改變 score 的值
```

在 TypeScript 中，**預設優先使用 `const`**，需要重複賦值時再用 `let`。

---

## 步驟 2：型別註解（`: 型別`）

在變數名稱後加上 `: 型別`，告訴編譯器這個變數允許的值。

```typescript
const userName: string = "小明";
const age: number = 18;
const passed: boolean = true;
```

- **`string`**：字串  
- **`number`**：數字（整數、浮點數皆為 `number`）  
- **`boolean`**：`true` 或 `false`  

若賦值的型別不符，編輯器會立刻標錯：

```typescript
// const wrong: number = '一百'  // ❌ string 不可賦給 number
```

---

## 步驟 3：型別推斷

若**宣告時就給定值**，TypeScript 常能自動推斷型別，此時可省略註解：

```typescript
const title = "Hello"; // 推斷為 string
const count = 10; // 推斷為 number
const ok = false; // 推斷為 boolean
```

**何時建議寫明型別？**

- 先宣告、晚一點才賦值  
- 希望限制「只能是某幾種值」時（與後續聯合型別、字面量型別搭配）  
- 團隊規範要求可讀性與 API 邊界清楚  

```typescript
let total: number;
total = 0;
total = total + 1;
```

---

## 步驟 4：`bigint`（選讀）

當整數**超過 `number` 的安全整數範圍**，可使用 `bigint`：

```typescript
const big: bigint = 9007199254740993n;
```

一般教學與前端專案仍以 `number` 為主；遇到金額、大整數運算再查 `bigint` 即可。

## 步驟 5：在練習檔中試跑完整範例

在**練習檔**末尾（或覆寫前面實驗碼）貼上並修改：

```typescript
const appName = "TS 練習";
let visitCount: number = 0;

function tick() {
  visitCount = visitCount + 1;
  console.log(`${appName}：第 ${visitCount} 次`);
}

tick();
tick();
```

執行 `npm run dev`，在頁面上操作「造訪 +1」等元件觀察 `let` 與畫面連動（與 tsx-cli 的 `tick()` 示範觀念相同，呈現方式不同）。

## 重點整理

| 概念 | 說明 |
|------|------|
| `const` / `let` | 優先 `const`；需要重複賦值用 `let` |
| `: string` 等 | 明確型別註解，便於檢查與閱讀 |
| 型別推斷 | 有初始值時常可省略註解 |
| `string`、`number`、`boolean` | 最常用的三種基本型別 |
