## 步驟 1：陣列型別的兩種寫法

元素型別後加 **`[]`**，或使用泛型 **`Array<元素型別>`**，兩者等價：

```typescript
const nums: number[] = [1, 2, 3]
const tags: Array<string> = ['ts', 'vite']

const names: string[] = ['小明', '小華']
// names.push(1)  // ❌ number 不是 string
```

**空陣列**若未註解，可能被推斷成 `never[]`，建議寫明型別：

```typescript
const scores: number[] = []
scores.push(95)
```

---

## 步驟 2：物件陣列

搭配 `interface` / `type` 描述「陣列裡每一筆」的形狀：

```typescript
interface Task {
  id: number
  title: string
}

const tasks: Task[] = [
  { id: 1, title: '寫文件' },
  { id: 2, title: '寫測試' },
]
```

---

## 步驟 3：唯讀陣列

若希望**不要透過索引或方法改動陣列內容**（仍可能複製後再改，語意是「這個參考當唯讀使用」），可使用：

```typescript
const seq: readonly number[] = [1, 2, 3]
// seq.push(4)  // ❌ 唯讀陣列沒有 push（視設定而定）

// 或使用 ReadonlyArray<number>
const labels: ReadonlyArray<string> = ['a', 'b']
```

與 `const` 變數的差異：`const arr = []` 仍可能 `arr.push(...)`；加上 **`readonly`** 是對**陣列操作**多一層限制。

---

## 步驟 4：泛型是什麼？

**泛型**（Generics）讓型別可以**先留一個占位符 `T`**，在使用時再決定具體型別，達到「同一段邏輯、多種型別都安全」。

內建範例：`Array<string>` 就是「`Array` 這個型別建構子」配上 **`string`**。

```typescript
function firstItem<T>(items: T[]): T | undefined {
  return items[0]
}

const n = firstItem([10, 20, 30])      // T 推斷為 number
const s = firstItem(['a', 'b'])        // T 推斷為 string
```

- **`<T>`**：宣告「這裡有一個型別參數叫 T」  
- 傳入的陣列元素型別會**推斷** T，回傳與元素同型別或 `undefined`

---

## 步驟 5：常見泛型：`Promise<T>`

非同步函式若回傳 Promise，可標註**解析後**的型別：

```typescript
async function loadId(): Promise<number> {
  return 42
}

const id = await loadId()  // id 為 number
```

前端呼叫 API 時常寫成 `Promise<Response>` 或自訂 `Promise<User>` 等。

---

## 步驟 6：`Record`（選讀）

以**字串鍵**對應到**同一種值型別**時，可用 `Record`：

```typescript
type Locale = 'zh' | 'en'

const greetings: Record<Locale, string> = {
  zh: '你好',
  en: 'Hello',
}
```

進階用法可與 [範例 5](範例5-聯合型別與型別窄化.md) 的聯合型別一起讀。

## 步驟 7：綜合練習（在練習檔中）

在**練習檔**（`tsx-cli/src/04-arrays-generics.ts` 或 `examples/…/src/practice-04-arrays-generics.ts`）末尾加入（或覆寫前面實驗碼）：

```typescript
interface Score {
  name: string;
  value: number;
}

function average(scores: number[]): number {
  if (scores.length === 0) return 0;
  const sum = scores.reduce((a, b) => a + b, 0);
  return sum / scores.length;
}

function mapNames(rows: Score[]): string[] {
  return rows.map((r) => r.name);
}

const data: Score[] = [
  { name: "國文", value: 80 },
  { name: "數學", value: 92 },
];

console.log(average(data.map((d) => d.value)));
console.log(mapNames(data));
```

執行 `npm run dev`，在瀏覽器**開發者工具 → Console** 或頁面上「頁面輸出」區塊查看輸出。

## 重點整理

| 寫法 | 說明 |
|------|------|
| `number[]` / `Array<number>` | 數字陣列，兩種寫法等價 |
| `Task[]` | 自訂型別的陣列 |
| `readonly T[]` / `ReadonlyArray<T>` | 唯讀陣列 |
| `<T>(...) => ...` | 函式泛型 |
| `Promise<T>` | 非同步完成後得到 T |
