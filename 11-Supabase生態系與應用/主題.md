# 11 - Supabase 生態系與應用

## 主題說明

本單元以 **Next.js (App Router)** 為基礎，介紹如何整合 **Supabase** (目前最受歡迎的開源 Firebase 替代方案之一)。

我們將帶領你從一個乾淨的專案開始，逐步擴充各項功能。透過這個過程，你將學習到如何運用完整的 BaaS (Backend as a Service) 解決方案，建立具備身分驗證、關聯式資料庫、雲端儲存與即時功能的現代全端應用程式。

---

### 學習目標

- **專案設定**：學習如何建立 Supabase 專案並在 Next.js 中整合 `@supabase/ssr` 官方 SDK。
- **Authentication (身分驗證)**：實作 Email 註冊登入與第三方登入 (Google/GitHub)，並了解 Cookie-based session 管理。
- **Database & RLS (資料庫與權限控制)**：學習建立 PostgreSQL 資料表，透過 UI 與 SQL 操作資料，並設定 Row Level Security (RLS) 以確保資料安全。
- **Storage (雲端儲存)**：實作檔案/圖片的上傳、下載與刪除功能。
- **Realtime (即時更新)**：透過 WebSocket 訂閱資料庫變動，實作即時畫面更新功能。

---

### 練習專案建議（一票玩到底，逐步擴充）

本章的 5 個範例以及最終的整合實戰，都建議在**同一個 Next.js 專案**中完成。每一課都會在前一課的基礎上增加新功能，最終組裝成一個完整的應用程式。

> 先在終端機執行以下指令建立專案：

```bash
npx create-next-app@latest supabase-fullstack-app --typescript --tailwind --app --src-dir --no-import-alias
cd supabase-fullstack-app
```

進入專案目錄後，安裝 Supabase 所需套件：

```bash
npm install @supabase/supabase-js @supabase/ssr
```

### 學習成果檢核（完成後你應該能做到）

- 能夠獨立在 Supabase 後台建立專案、獲取 API 憑證，並將專案與 Next.js 進行連線。
- 掌握 Next.js 伺服器端元件 (Server Components) 與客戶端元件 (Client Components) 之間如何安全地傳遞與存取 Supabase 資料。
- 理解並實作 Row Level Security (RLS)，確保使用者只能存取屬於自己的資料。
- 具備上傳檔案至 Storage Bucket 並獲取公開網址的實作能力。
- 能夠建立具備即時互動特性 (Realtime) 的現代化網頁應用程式。
