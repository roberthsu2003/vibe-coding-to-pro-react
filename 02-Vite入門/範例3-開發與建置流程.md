# 範例 3：開發與建置流程

## 目標

理解 `npm run dev`、`npm run build`、`npm run preview` 的差異與使用時機。

---

## 步驟 1：開發模式（dev）

```bash
npm run dev
```

**用途**：啟動開發伺服器，供日常開發使用。

**特點**：

- 啟動快
- 改程式後自動熱更新
- 預設網址：http://localhost:5173
- 產出的程式**未壓縮**，檔案較大，不適合部署

---

## 步驟 2：建置模式（build）

```bash
npm run build
```

**用途**：產生可部署的生產版本。

**特點**：

- 輸出到 `dist/` 資料夾
- 程式會壓縮、優化
- 適合上傳到伺服器或靜態託管（如 Vercel、Netlify）

---

## 步驟 3：預覽模式（preview）

```bash
npm run preview
```

**用途**：在本地預覽建置後的結果。

**使用時機**：執行 `npm run build` 之後，想確認部署前的畫面是否正確。

---

## 步驟 4：流程對照

| 階段 | 指令 | 輸出 |
|------|------|------|
| 開發 | `npm run dev` | 本地伺服器，即時更新 |
| 建置 | `npm run build` | `dist/` 資料夾 |
| 預覽 | `npm run preview` | 本地預覽 dist 內容 |

---

## 完成

掌握這三個指令，就能完成從開發到部署的流程。
